# Flask Routing

Flask App Routing is the process of mapping URLs to specific functions in a web application. When a user enters a URL in the browser, Flask checks the defined routes and executes the corresponding function to generate a response.

This is done using the `@app.route()` decorator, which binds a URL path to a function.

---

# Basic Routing Example

In this example, we define two routes:

- `/` for the homepage
- `/hello` for displaying a welcome message

Each route is linked to a function using the `@app.route()` decorator.

```python
from flask import Flask
app = Flask(__name__)

@app.route("/hello")
def hello():
    return "Hello, Welcome to SETEC"

@app.route("/")
def index():
    return "Homepage of SETEC"

if __name__ == "__main__":
    app.run(debug=True)
```

## Explanation

- `@app.route("/hello")` maps the URL `/hello` to the `hello()` function.
- `@app.route("/")` maps the homepage URL `/` to the `index()` function.
- `debug=True` enables debug mode for development.

## Run the Application

```bash
python filename.py
```

## Output

Open the browser and visit:

```text
127.0.0.1:5000/hello
```

You will see:

```text
Hello, Welcome to SETEC
```

# Dynamic URLs

We can build dynamic URLs using variables in the route.

To add variables to URLs, use:

```python
<variable_name>
```

The function receives the value as a parameter.

## Example

```python
from flask import Flask

app = Flask(__name__)

@app.route('/user/<username>')
def show_user(username):
    return f'Hello {username} !'

@app.route("/hello")
def hello():
    return "Hello, Welcome to SETEC"

@app.route("/")
def index():
    return "Homepage of SETEC"

if __name__ == "__main__":
    app.run(debug=True)
```

## Output

Open the browser and visit:

```text
127.0.0.1:5000/user/តារា
```

Result:

```text
Hello តារា !
```

# URL Converters

By default, variables are treated as strings.

You can specify data types using:

```python
<converter:variable_name>
```

## Supported Converters

| Converter | Description                                |
| --------- | ------------------------------------------ |
| `string`  | Default type, accepts text without slashes |
| `int`     | Accepts positive integers                  |
| `float`   | Accepts floating-point numbers             |
| `path`    | Accepts strings including slashes          |
| `uuid`    | Accepts UUID strings                       |

---

# Converter Example

```python
from flask import Flask

app = Flask(__name__)

@app.route('/post/<int:id>')
def show_post(id):
    return f'This post has the id {id}'

@app.route('/user/<username>')
def show_user(username):
    return f'Hello {username} !'

@app.route("/hello")
def hello():
    return "Hello, Welcome to SETEC"

@app.route("/")
def index():
    return "Homepage of SETEC"

if __name__ == "__main__":
    app.run(debug=True)
```

## Output

Visit:

```text
127.0.0.1:5000/post/13
```

Result:

```text
This post has the id 13
```

# add_url_rule() Function

Flask also allows URL mapping using the `add_url_rule()` function.

This is useful when the view function is defined in another module.

The `@app.route()` decorator internally calls `add_url_rule()`.

## Syntax

```python
add_url_rule(rule, endpoint, view_func)
```

### Parameters

| Parameter   | Description          |
| ----------- | -------------------- |
| `rule`      | URL pattern          |
| `endpoint`  | Name of the endpoint |
| `view_func` | Function to execute  |

---

# Example Using add_url_rule()

```python
from flask import Flask
app = Flask(__name__)

def show_user(username):
    return f'Hello {username} !'

app.add_url_rule('/user/<username>', 'show_user', show_user)

if __name__ == "__main__":
    app.run(debug=True)
```

## Output

Visit:

```text
127.0.0.1:5000/user/សុខា
```

Result:

```text
Hello សុខា !
```
